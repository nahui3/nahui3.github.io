# План: nahui3.github.io

**Назначение:** Документация проекта и BDUI: архитектура, фичи и манифесты, API backend, использование библиотек nahui3 на iOS, Android и Web.

## Контекст

- Фича — небольшой проект со своим манифестом и набором файлов-узлов (id от пути, имя файла, хеш). Манифест — дерево файлов и папок; формат XML, JSON, YAML или другой. Корневая фича перечисляет остальные фичи и их расположение.
- Клиент передаёт версию платформы и фичу; сервер отдаёт дельту или «нет изменений». Дельта строится и кешируется только на сервере.
- Платформы: iOS — SwiftUI SPM nahui3, Android — Kotlin Compose nahui3, Web — React nahui3.

## Задачи

1. **Контент**
   - Раздел «BDUI»: обзор архитектуры (админка, CI/CD-воркер, backend, клиенты), понятия фичи, манифеста, узла (id, имя файла, хеш), корневой фичи, версии платформы и накатанной версии.
   - Описание потока: клиент → версия платформы + id фичи → сервер сравнивает манифесты, отдаёт дельту или «нет изменений»; дельта только в кеше сервера.
   - Описание API: эндпоинты запроса фичи (полный набор + манифест или дельта), параметры (платформа, id фичи, версия платформы).
   - Краткие инструкции по использованию библиотек nahui3 (ios/android/web) для загрузки фичи, передачи версии и применения дельты.

2. **Ссылки**
   - Обновить список репозиториев и ссылки на backend, admin, ios, android, web, demo; при необходимости раздел «Референсы» (DivKit, LiveView, Turbo).

3. **Единообразие**
   - Стили и навигация в соответствии с текущим index.html/styles.css.

## Результат

Разработчики и продукт могут прочитать на сайте, как устроен BDUI, что такое фича и манифест, как работает версионирование и дельта, и как интегрировать библиотеки nahui3.
