План: Admin

Управление BDUI: настройка веток проекта для CI/CD-воркера, просмотр версий по платформам, выбор версии для публикации, фиксация версий у клиентов.

Контекст

  • На бэкенде в CI/CD работает воркер, который периодически (по таймеру или при пуше в ветку) анализирует указанные в настройках админки ветки проекта и на основе входящих файлов строит дерево хешей и манифесты, версионирует их.
  • В админке можно посмотреть любую версию для каждой платформы и указать, какую версию нужно опубликовать и какие версии у клиентов.

Задачи

  1. Настройки для воркера — хранение и редактирование настроек: репозитории (или пути к .git), ветки для анализа; связь с backend/воркером.
  2. Просмотр версий — список версий по платформам (ios, android, web), построенных воркером; просмотр содержимого манифеста (корневая фича, дерево папок и файлов с id, именем, хешем).
  3. Выбор версии для публикации — указание опубликованной версии для каждой платформы; сохранение выбора в хранилище, доступном backend.
  4. Выбор версий у клиентов — указание версий, находящихся у клиентов для каждой платформы; только по этим версиям сервер может строить дельту.
  5. Интеграция с файлами — при необходимости оставить текущий поток загрузки файлов (InterfaceFile) для статичных ресурсов; основной поток версионирования — через воркер и манифесты.
Результат: Админка — точка настройки источника изменений, просмотра версий и выбора версии для публикации и у клиентов по платформам.
